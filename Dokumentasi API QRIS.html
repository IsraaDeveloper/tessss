Pendahuluan

Selamat datang di dokumentasi API Payment Gateway kami. Dokumen ini akan memandu Anda dalam mengintegrasikan sistem Anda dengan layanan kami di sanpay.site.

Langkah Awal Integrasi

  1. Login ke dashboard, lalu masuk ke halaman Setting API.
  2. Generate API Key dan Merchant Code Anda.
  3. Masukkan URL endpoint Anda pada kolom URL Callback, lalu gunakan tombol Validasi URL untuk memastikan server kami bisa menjangkau server Anda.
PENTING: Whitelist IP Address

Untuk memastikan server Anda dapat menerima notifikasi dari API Callback kami, Anda wajib memasukkan IP Address server kami ke dalam daftar whitelist di firewall Anda:

103.127.137.140

API Generate QRIS

Endpoint ini digunakan untuk membuat QRIS Dinamis berdasarkan nominal yang Anda kirim. Sistem kami akan membuatkan `qrContent` yang bisa Anda tampilkan sebagai gambar QR.

Endpoint

POST https://sanpay.site/api/v1/topup_qris

Validasi Keamanan

Setiap request ke endpoint ini wajib menyertakan 2 header khusus untuk validasi keamanan:

Header Keterangan
X-Merchant-Code Merchant Code unik Anda.
X-Signature Tanda tangan digital (signature) dari payload (raw body JSON) yang dikirim. Dibuat menggunakan algoritma HMAC-SHA256 dengan API Key Anda sebagai kunci rahasianya.

Struktur Body Request (JSON)

{
  "amount": 50000,
  "partnerReferenceNo": "INV-2025-09-00123",
  "expirySeconds": 300
}
Key Tipe Deskripsi
amount Integer Wajib. Nominal transaksi dalam bentuk bilangan bulat.
partnerReferenceNo String Wajib. ID unik transaksi dari sisi Anda (contoh: nomor invoice). Harus unik. max 64 Karakter
expirySeconds Integer Opsional. Durasi validitas QRIS dalam detik. Jika tidak diisi, default adalah 900 detik (15 menit).

Contoh Respons Sukses (JSON)

{
  "status": "success",
  "partnerReferenceNo": "INV-2025-09-00123",
  "merchantName": "Nama Merchant Anda",
  "amount": 50000,
  "qrContent": "000201010212266....DBD7",
  "expiresAt": "2025-09-16 10:45:00"
}

API Topup Virtual Account

Endpoint ini digunakan untuk membuat Virtual Account (VA) untuk berbagai bank.

Endpoint

POST https://sanpay.site/api/v1/topup_va

Validasi Keamanan

Sama seperti API QRIS, wajib menyertakan header X-Merchant-Code dan X-Signature.

Struktur Body Request (JSON)

{
  "amount": 50000,
  "partnerReferenceNo": "INV-VA-001",
  "bank_code": "BCA",
  "name": "John Doe"
}
Key Tipe Deskripsi
amount Integer Wajib. Nominal transaksi.
partnerReferenceNo String Wajib. ID unik transaksi Anda.
bank_code String Wajib. Kode Bank (BCA, BNI, BRI, MANDIRI, PERMATA, BSI).
name String Opsional. Nama pelanggan.

Contoh Respons Sukses (JSON)

{
  "status": "success",
  "partnerReferenceNo": "INV-VA-001",
  "amount": 50000,
  "bank_code": "BCA",
  "va_number": "1234567890",
  "expiration_date": "2025-09-16T10:45:00+07:00"
}

API Get Channels

Endpoint ini digunakan untuk mengambil daftar channel VA (Virtual Account) dan Retail (Alfamart/Indomaret) yang tersedia dan aktif, beserta informasi biaya admin untuk setiap channel.

Endpoint

GET https://sanpay.site/api/v1/get_channels

Parameter

Parameter Keterangan
apikey Wajib. API Key dari dashboard Anda.
merchant_code Wajib. Merchant Code dari dashboard Anda.

Contoh URL Request

https://sanpay.site/api/v1/get_channels?apikey=APIKEY_ANDA&merchant_code=MERCHANT_CODE_ANDA

Contoh Respons Sukses (JSON)

{
  "status": "success",
  "data": {
    "va_channels": [
      {
        "code": "BCA",
        "name": "Bank Central Asia",
        "min_amount": 10000,
        "max_amount": 50000000,
        "admin_fee_type": "FIXED",
        "admin_fee_value": 4000
      },
      {
        "code": "BNI",
        "name": "Bank Negara Indonesia",
        "min_amount": 10000,
        "max_amount": 0,
        "admin_fee_type": "PERCENT",
        "admin_fee_value": 1.5
      }
    ],
    "retail_channels": [
      {
        "code": "ALFAMART",
        "name": "Alfamart",
        "min_amount": 10000,
        "max_amount": 5000000,
        "admin_fee_type": "FIXED",
        "admin_fee_value": 2500
      },
      {
        "code": "INDOMARET",
        "name": "Indomaret",
        "min_amount": 10000,
        "max_amount": 5000000,
        "admin_fee_type": "FIXED",
        "admin_fee_value": 2500
      }
    ]
  }
}

Penjelasan Field Response

Field Tipe Deskripsi
code String Kode channel (contoh: "BCA", "BNI", "ALFAMART")
name String Nama lengkap channel
min_amount Integer Nominal minimum transaksi (dalam Rupiah)
max_amount Integer Nominal maksimum transaksi (dalam Rupiah). Jika bernilai 0 berarti unlimited (tidak ada batas)
admin_fee_type String Jenis biaya admin:
• "FIXED" = Biaya tetap dalam Rupiah
• "PERCENT" = Biaya persentase dari nominal transaksi
admin_fee_value Float Nilai biaya admin:
• Jika admin_fee_type = "FIXED", nilai dalam Rupiah (contoh: 4000 = Rp 4.000)
• Jika admin_fee_type = "PERCENT", nilai dalam persen (contoh: 1.5 = 1.5%)
Contoh Perhitungan Biaya Admin:
  • FIXED: Jika transaksi Rp 100.000 dengan admin_fee_type = "FIXED" dan admin_fee_value = 4000, maka total yang harus dibayar = Rp 100.000 + Rp 4.000 = Rp 104.000
  • PERCENT: Jika transaksi Rp 100.000 dengan admin_fee_type = "PERCENT" dan admin_fee_value = 1.5, maka total yang harus dibayar = Rp 100.000 + (1.5% × Rp 100.000) = Rp 100.000 + Rp 1.500 = Rp 101.500

API Callback (Webhook)

API Callback adalah mekanisme notifikasi otomatis dimana sistem kami akan secara proaktif "memberi tahu" server Anda setiap kali ada pembayaran yang berhasil.

Validasi Endpoint Callback Anda

Di halaman Setting API, terdapat tombol "Validasi URL". Tombol ini akan mengirimkan sebuah request tes ke URL yang Anda masukkan. Tujuannya adalah untuk memastikan server Anda dapat dijangkau dan merespons dengan benar.

Agar validasi berhasil, server Anda harus:

  1. Menerima request POST dari kami.
  2. Memvalidasi header `X-Merchant-Code` dan `X-Signature` yang kami kirim (opsional untuk tes, tapi wajib untuk transaksi sungguhan).
  3. Merespons dengan HTTP Status Code 200 OK.

Berikut adalah contoh payload JSON yang akan kami kirim saat Anda menekan tombol validasi:

{
  "transactionID": "TEST_1726456800",
  "amount": 1,
  "customerName": "VALIDATION TEST",
  "isValidationTest": true
}

Validasi Keamanan Callback

Setiap request callback dari server kami akan menyertakan 2 header khusus untuk validasi keamanan:

Header Keterangan
X-Merchant-Code Merchant Code unik Anda.
X-Signature Tanda tangan digital (signature) dari payload yang dikirim. Dibuat menggunakan algoritma HMAC-SHA256 dengan API Key Anda sebagai kunci rahasianya.

Anda wajib memvalidasi kedua header ini untuk memastikan notifikasi berasal dari sumber yang sah.

Struktur Body Request (JSON)

Berikut adalah detail dari data JSON yang akan kami kirimkan ke URL callback Anda.

{
  "transactionID": "QRS2025091512345",
  "merchantId": "MERCHANT001",
  "storeId": "STORE001",
  "amount": 10000,
  "transactionDate": "2025-09-15 10:30:00",
  "customerName": "John Doe",
  "referenceNo": "REF12345678"
}
Key Tipe Deskripsi
transactionID String ID unik transaksi dari sistem kami.
merchantId String ID merchant yang terkait dengan transaksi.
storeId String ID toko/outlet yang terkait dengan transaksi.
amount Integer Jumlah nominal transaksi dalam bentuk bilangan bulat.
transactionDate String Waktu transaksi dalam format YYYY-MM-DD HH:MM:SS.
customerName String Nama pelanggan yang melakukan pembayaran.
referenceNo String Nomor referensi. Nilai akan muncul jika Api Generete QRIS digunakan yang menyertakan partnerReferenceNo, nilai akan kosong jika tidak membuat sistem untuk Api Generete QRIS. (tidak wajib)

Struktur Callback untuk Virtual Account & Retail

Untuk pembayaran melalui Virtual Account atau Retail (Alfamart/Indomaret), struktur callback yang dikirim ke server Anda adalah sebagai berikut:

{
  "status": "success",
  "partnerReferenceNo": "INV-VA-001",
  "external_id": "VA_1732089600_abc123",
  "amount": 50000,
  "payment_status": "PAID"
}
Key Tipe Deskripsi
status String Status callback, selalu "success" untuk pembayaran berhasil.
partnerReferenceNo String ID referensi yang Anda kirimkan saat membuat transaksi.
external_id String ID unik transaksi dari sistem kami.
amount Integer Nominal yang dibayarkan.
payment_status String Status pembayaran, nilai "PAID" untuk pembayaran berhasil.

Contoh Kode PHP Penerima & Validator

<?php
$your_merchant_code = 'MC-xxxxxxxx';
$your_api_key = 'APIKEY_RAHASIA_ANDA';

$merchant_code_header = $_SERVER['HTTP_X_MERCHANT_CODE'] ?? '';
$signature_header = $_SERVER['HTTP_X_SIGNATURE'] ?? '';

$raw_body = file_get_contents('php://input');

if ($merchant_code_header !== $your_merchant_code) {
    http_response_code(401);
    echo json_encode(['status' => 'error', 'message' => 'Invalid Merchant Code']);
    exit;
}

$calculated_signature = hash_hmac('sha256', $raw_body, $your_api_key);
if (!hash_equals($calculated_signature, $signature_header)) {
    http_response_code(401);
    echo json_encode(['status' => 'error', 'message' => 'Invalid Signature']);
    exit;
}

$data = json_decode($raw_body, true);

http_response_code(200);
echo json_encode(['status' => 'success']);
?>

API Get Mutasi

Gunakan API ini untuk mengambil riwayat 50 transaksi terakhir secara manual. Berguna untuk rekonsiliasi atau pengecekan cepat.

Endpoint

GET https://sanpay.site/api/v1/get_mutasi

Parameter

Parameter Keterangan
apikey Wajib. API Key dari dashboard Anda.
merchant_code Wajib. Merchant Code dari dashboard Anda.

Contoh URL Request

https://sanpay.site/api/v1/get_mutasi?apikey=APIKEY_ANDA&merchant_code=MERCHANT_CODE_ANDA

Contoh Respons Sukses

{
  "status": "success",
  "message": "Data mutasi berhasil diambil",
  "data": [
    {
      "transactionID": "QRS2025091512345",
      "transactionDate": "2025-09-15 10:30:00",
      "amount": 10000,
      "customerName": "John Doe"
    }
  ]
}
